## Code of Conduct

By participating in this project, you agree to abide by our Code of Conduct:
- Be respectful and inclusive to all contributors
- Provide constructive feedback
- Focus on the project's goals and quality
- Avoid discriminatory or offensive language

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/infinity-gym.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Commit your changes: `git commit -m "Description of your changes"`
6. Push to your fork: `git push origin feature/your-feature-name`
7. Create a pull request

## How to Contribute

You can contribute to Infinity Gym in several ways:

1. **Bug Reports**: Report bugs or issues you encounter
2. **Feature Requests**: Suggest new features or improvements
3. **Code Contributions**: Implement new features or fix bugs
4. **Documentation**: Improve existing documentation or create new guides
5. **Testing**: Help improve test coverage or write new tests

## Development Process

1. Ensure you have Node.js installed
2. Install dependencies: `npm install`
3. Run tests to ensure everything works: `npm test`
4. Make your changes
5. Write tests for your changes if applicable
6. Run tests again to ensure nothing is broken
7. Update documentation if needed

## Style Guidelines

### HTML/CSS
- Follow semantic HTML practices
- Use consistent indentation (2 spaces)
- Use meaningful class and ID names
- Follow mobile-first responsive design principles

### JavaScript
- Use ES6+ features where appropriate
- Follow camelCase naming convention
- Use descriptive variable and function names
- Comment complex logic
- Keep functions focused and small

### General
- Maintain consistency with existing code
- Write clean, readable code
- Remove unused code and dependencies
- Follow accessibility best practices

## Reporting Issues

When reporting issues, please include:
1. A clear and descriptive title
2. Steps to reproduce the issue
3. Expected behavior
4. Actual behavior
5. Screenshots if applicable
6. Browser and operating system information
7. Any error messages in the console

## Pull Request Process

1. Ensure your code follows the style guidelines
2. Update documentation if you've changed functionality
3. Add tests for new functionality
4. Ensure all tests pass
5. Describe your changes in the pull request
6. Reference any related issues
7. Be prepared to make revisions based on feedback

### Pull Request Checklist
- [ ] Code follows style guidelines
- [ ] Documentation has been updated
- [ ] Tests have been added/updated
- [ ] All tests pass
- [ ] Branch is up to date with main
- [ ] Changes have been reviewed locally

## Questions?

If you have any questions about contributing, feel free to open an issue or contact the maintainers.

Thank you for helping make Infinity Gym better!
